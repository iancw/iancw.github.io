As a team lead, I'll be participating in the interview process. We've got an interview tomorrow, and I've been asked to prepare to spend some time with the candidate. So what questions are good to ask when interviewing a Java developer?

  • When is a finally block entered?
    • After catching an exception?
    • After successful completion of the try block?
    • Will it be entered if the exception is not caught?
    • Will it be entered if the application exits?
  • Can you pass by value in Java?
    (Answer: primitives are passed by value, you could argue that references=pointers, and that the pointers are passed by value)
  • Is there a difference between Hashtable and HashMap?
  • How about between Vector and ArrayList?
  • What's the difference between swing and awt?
    (Answer: Light-weight vs heavy-weight)
    • What's the difference between light and heavy weight components?

      • (Answer: Light-weight = same appearance on all platforms, but slower execution)
      (AWT components map to the OS's windowing toolkit, whereas swing is implemented with Java2d)

    • Are light and heavy weight components interchangable?
      (Looking for knowledge that they shouldn't generally be used together due to z-buffer incompatabilities)
    • Is there a naming convention to distinguish heavy from lightweight components?
      (the J-prefix...)
  • What are the values of s1 and s2 after invoking the swap method?
    public void swap(String s1, String s2)
    {
    String tmp = s1;
    s1 = s2;
    s2 = tmp;
    }

    String s2 = "b";
    swap(s1, s2);

    • What happens if we make the arguments of the swap function final?
